# ADR 0003: 히어로 v2 — Bleed + 4단 Blur + Ken Burns 채택

**상태**: 채택
**날짜**: 2026-05-24
**관련**: [domains/home.md](../domains/home.md)
**종결**: ADR 0002 미결 "히어로 인터랙티브 Option 1 vs 3"

## 배경

ADR 0002 시점에 Option 1·3 외 7개 변형(`__sketches/home-hero/`) 검토 후, 사진 없는 베이스라인(텍스트 + 블롭)으로 이동했다가 다시 사진 도입 방향으로 재검토했다. `__sketches/home-hero-v2/` 에서 총 14개 시안을 거쳐 12번(Stable Bleed + Text)을 채택한다.

진행 중 사용자 피드백으로 좁혀진 핵심 제약:

1. 사진이 히어로의 정확한 절반을 갈라 차지하는 "카드/사각형 경계" 느낌이 싫다.
2. 좌측 경계만 자연스럽게 흐릿하게 페이드되는 형태(초기 5번 안)는 좋다 — 단 직선 마스크는 그것마저 경계로 느껴진다.
3. 이미지·문구가 좌우로 **이동**하는 인터랙션은 별로다 — 위치가 바뀔 땐 페이드 아웃/인으로.
4. CTA가 스크롤마다 움직이는 것이 거슬린다 — 보여줄 거면 위치 고정.
5. 10번의 blur(0→18→0) + Ken Burns 디졸브가 아주 좋다.
6. 6번의 도트 진행 안내(스크롤에 따른 단계 안내)가 좋다.

## 결정

**히어로는 사진 4단 Bleed + Blur 페이드 + Ken Burns(sketch 12)로 구성한다.**

- 4단계 스크롤 히어로 (360vh): 외관 → 내부 → 예배 → 담임목사
- 사진: 우측 82% 폭, 사방으로 화면 밖 흘러넘김 (사각형 경계 안 보임), 좌측만 linear-gradient 마스크로 부드러운 페이드
- 전환: blur(0→20→0) + scale + 활성 사진 9초 Ken Burns
- 텍스트: 같은 위치에서 blur 페이드로 4단 변화
- CTA·위치: 완전 고정
- 상단 도트 진행 바
- 모바일: 사진을 더 넓게 깔되 불투명도 0.35~0.5로 낮춰 배경처럼, 텍스트는 그 위에 풀폭으로

## 결과 / 영향

### 직접 영향

- ADR 0002 미결 **"히어로 인터랙티브: Option 1 vs Option 3"** 종결.
- [domains/home.md](../domains/home.md) "1. HomeHero v2" 갱신 + "히어로 인터랙티브" 섹션 종결 + 미결 항목 정리.

### 코드 변경

- `src/domains/home/data/heroStages.ts` 신설 (`HERO_STAGES` 4단).
- `src/domains/home/components/widgets/HomeHero/index.tsx` v2로 교체 (sticky 360vh, blur 디졸브, Ken Burns).
- `src/app/globals.css` 에 `.b1-hero-kenburns` / `.b1-hero-scroll-drop` / `.b1-hero-ph-*` 추가.

### 후속 / 잔여 미결

- 본당 내부·예배 풍경·담임목사 정면 사진 3장 촬영 필요 (미확보 시 그라디언트 자리표시자로 동작).
- 4단 카피는 전부 placeholder — 담임목사·사무국 검수. 0번 교단·창립 연도, 3번 담임목사 이름·부임 연도 실값 확인.
- 2단 본문 데이터("5월 24일 주일 · 이사야 41장") 매주 갱신 운영 정책 결정 (정적 상수 PR vs CMS).

### `__sketches/` 처리

9 + 14개 시안은 삭제하지 않고 보존 (`__sketches/`는 일회성·빌드 미포함). 향후 다른 페이지 베이스라인 + 본 ADR 인용 근거 유지.
